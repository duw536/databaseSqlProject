# 온라인 서점 데이터베이스 모델링 프로젝트

1. 프로젝트 개요
대용량 트랜잭션이 발생하는 온라인 서점의 핵심 업무인 '회원-주문-도서' 프로세스를 분석하여, 데이터 무결성을 보장하는 관계형 데이터베이스를 구축했습니다.
목표: 다대다 관계 해소 및 제3정규형을 적용한 논리적/물리적 모델링 구현
환경: Oracle DBMS, SQL Developer

2. 데이터베이스 구조
핵심 엔티티 6개를 도출하여 설계를 수행했습니다.
MEMBER	회원 정보 및 적립금 관리	
ORDERS	주문 헤더 (배송지, 총액)	FK: 회원ID
ORDER_DETAIL	주문 상세 (개별 도서 수량/가격)	N:M 관계 해소 (약한 엔티티)
BOOK	도서 정보 및 재고 관리	FK: 카테고리, 출판사
CATEGORY	도서 분류 코드	정규화 적용
PUBLISHER	출판사 정보	정규화 적용

3. 주요 구현 내용
33개의 시나리오를 SQL로 구현하여 검증했습니다.
모델링: 제3정규형 준수, 인덱스 설계를 통한 조회 성능 최적화

4. 실행 방법
스크립트 파일 내용만 순서대로 실행하면 데이터베이스가 만들어질 수 있도록 작성했습니다.
